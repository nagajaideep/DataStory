Noa Voice Agent: Building Sub-Second Calling for Therapy

How we architected real-time voice calling with Rumik AI, optimized latency from 2.5s to 300ms, and stretched $20K credits across 10,000 users.

10 min read · By Anshap Engineering

Introduction

Building a voice calling feature for mental health requires speed, empathy, and reliability. Noa isn't just a chatbot—she's a real-time voice companion that understands therapeutic context and responds with emotional intelligence.

In this story, we dissect the calling architecture, identify latency bottlenecks (we're at 830ms, but targeting 300ms), compare voice solutions (Google vs. Rumik Silk vs. in-house), and show how to optimize runway with smart API choices. With the right approach, $20,000 in GCP credits supports 10,000 users for a full year.

The Architecture: Brain vs. Voice

We chose a Hybrid Orchestration model. While all-in-one solutions exist, keeping Noa's "brain" separate from the voice "ears and mouth" allows us to maintain 100% ownership of the therapeutic logic.

01
User Input
Audio captured via Anshap App (WebSocket)
02
Google STT (v2)
Streaming conversion from voice to text (~250ms)
03
Noa Model (The Brain)
Core reasoning, therapy logic, and response generation
04
Google TTS
Neural2 streaming text-to-audio (~200ms)
05
Audio Out
Low-latency playback on user device

Why This approach?

By using Google Cloud Speech APIs for the voice layer, we leverage world-class infrastructure while keeping our core IP—the Noa Model—completely isolated. Google never see our model weights or therapeutic prompts.

Evolution: From Bot to Caller

We're transitioning from a simple request-response model to full-duplex real-time calling. This means moving the Main Model to the center, processing audio streams instead of waiting for silence. The result: a voice agent that feels like a real conversation.

Safety & Compliance

In mental health, safety is foundational. Our calling architecture includes:

  • Crisis Detection: Real-time intent detection for self-harm/ideation → human intervention.
  • Encrypted Transcripts: HIPAA-compliant encryption for all audio & text logs.
  • Session Memory: Firestore stores mood history for conversation continuity.
Phased Approach for Calling:
Phase 1: Ship with Google Speech APIs (proven, balanced)
Phase 2: Integrate Rumik Silk (sub-500ms, emotional)
Phase 3: Own in-house voice model (full control, cost-optimized)
